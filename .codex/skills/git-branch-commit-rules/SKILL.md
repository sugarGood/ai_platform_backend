# Git 提交与分支规范

## 提交前检查

- 先同步主干变更，尽早解决冲突。
- 提交前确保代码可编译、关键测试可运行。
- 清理无用 import、无用变量、调试代码。
- 检查本次提交说明是否准确表达变更内容。

## 分支模型

- `master/main`：生产分支，只接受合并。
- `develop`：主开发分支，汇总功能变更。
- `feature-*`：功能开发分支，从 `develop` 拉取并回合到 `develop`。
- `release-*`：发布分支，用于发布前稳定化。
- `hotfix-*`：线上修复分支，从生产分支拉取，修复后回合生产和开发分支。

## 命名建议

- 功能分支：`feature-{author}-{topic}`
- 修复分支：`hotfix-{topic}`
- 发布分支：`release-{version}`

## 合并建议

- 优先小步提交，减少一次性大合并。
- 合并前再次执行构建与测试。
- 关键分支禁止直接推送未验证代码。
