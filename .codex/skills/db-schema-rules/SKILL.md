# 数据库设计规范

## 建表基线

每张表建议具备：

- `id`（Long，自增）
- `create_time`（`datetime`，默认 `CURRENT_TIMESTAMP`）
- `update_time`（`datetime`，默认 `CURRENT_TIMESTAMP` 且自动更新时间）

## 字段设计

- 优先使用更小且合适的数据类型，避免过度放大。
- 固定长度字段优先 `char`，可变长度优先合适长度 `varchar`。
- 避免“能用数值却用字符串”的设计。

## 注释规范

- 枚举/状态字段注释必须写清楚所有取值含义。
- 修改状态定义时同步更新字段注释。

## 索引规范

- 每张表必须有主键索引。
- 按查询路径建设必要索引，避免冗余索引。
- 复合索引按最常见过滤条件顺序设计。
- 索引命名简洁明确（如 `<field>_index`）。

## 变更要求

- 结构变更需同步实体字段、注释和相关 Mapper。
- 变更前评估是否影响历史 SQL 与兼容性。
