# 交付检查清单

结束任务前执行这一份检查。

## 代码质量

- 改动遵循目标模块已有风格，没有引入第二套风格。
- 公共方法、接口、非显然分支在需要时具备准确注释。
- 没有留下过期注释。
- 没有无理由的超长参数列表。
- 没有把 servlet/web 传输对象无必要地泄漏进 service。

## 分层检查

- controller 仍然是薄层。
- service 负责业务流程。
- mapper/dao 命名符合 SQL 语义。
- entity/DTO 边界清晰。

## 事务与数据检查

- 新增或修改的 `@Transactional` 都有明确边界。
- 没有错误依赖同类内部调用触发事务。
- XML/SQL 没有把本该参数化的业务常量写死。
- 新增布尔字段命名符合 `*Flag` / `*_flag` 规则。

## 仓库整洁性

- 可行时，已对相关模块执行编译或测试。
- schema 变更已同步到相关测试夹具或 H2 schema。
- 已清理无用 import、死变量、调试代码。
- 没有引入 `target/`、IDE 文件、生成噪音或无关产物。

## Git 检查

- 变更范围仅覆盖当前任务。
- 提交前代码已格式化，改动说明清晰。
- 如果用户询问分支建议，优先使用：
  - `feature-{author}-{topic}` 处理功能开发
  - `hotfix-*` 处理线上紧急修复
  - release 分支只用于发版稳定
